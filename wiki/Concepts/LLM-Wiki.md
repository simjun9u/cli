# LLM-Wiki (대형 언어 모델 기반 개인 지식 베이스)

LLM-Wiki는 대형 언어 모델(LLM)을 개인 지식 관리(PKM) 및 디지털 정원 가꾸기의 컴파일러이자 정원사로 활용하는 지식 구축 패턴입니다. 기존 RAG(검색 증강 생성)의 한계를 극복하고, 지식이 단편화되어 사라지는 대신 시간이 흐를수록 복리로 누적되어 성장하는 **세컨드 브레인(Second Brain)**을 구축하는 것을 목적으로 합니다.

---

## 1. 핵심 철학: 정원사 비유와 복리 지식

* **RAG와의 차이**: 일반 RAG 시스템은 질문이 들어올 때마다 원본 문서들로부터 정보를 새롭게 찾아 조합합니다. 이는 질문할 때마다 지식을 매번 재발견하는 일회성 작업이며 지식이 누적되지 않습니다. 반면, LLM-Wiki는 새로운 정보가 들어올 때마다 이를 기존 위키 지식과 연결(Cross-reference)하고 합성하여 **지속적이고 영구적인 지식 구조**를 컴파일해 둡니다.
* **정원사 비유 (Gardener Metaphor)**: 사용자는 관심사나 가치가 담긴 원본 자료를 모으는 수집가 역할을 하고, LLM은 이를 정돈하여 위키에 심고 가지를 치며 교차 링크를 연결하는 정원사 역할을 수행합니다.
* **Gold In, Gold Out (목적성 있는 수집)**: 무지성으로 많은 양의 쓰레기 데이터를 모으는(Garbage In, Garbage Out) 대신, 사용자가 의미와 목적을 느끼고 선별한 양질의 데이터를 수집(Gold In)해야 고품질의 위키 아웃풋(Gold Out)이 생성됩니다.

---

## 2. 3계층 아키텍처 (Three-Layer Architecture)

LLM-Wiki는 세 가지 명확한 레이어로 구성됩니다.

```
┌──────────────────────────────────────────────┐
│                1. Raw Sources                │
│    (수정 불가능한 불변의 원본 정보 저장소)    │
│  - raw/ , raw_MyNote/ , Clippings/ , Bookmark/│
└──────────────────────┬───────────────────────┘
                       │ Ingest (LLM)
                       ▼
┌──────────────────────────────────────────────┐
│                 2. The Wiki                  │
│    (LLM이 작성하고 관리하는 지식 마크다운)   │
│  - wiki/ (Concepts, Dictionary, QnA, etc.)   │
└──────────────────────┬───────────────────────┘
                       │ Query & Lint (LLM)
                       ▼
┌──────────────────────────────────────────────┐
│                3. The Schema                 │
│      (에이전트의 작업 지침 및 메타 데이터)     │
│  - AGENTS.md (혹은 CLAUDE.md)                │
└──────────────────────────────────────────────┘
```

1. **Raw Sources (원본 자료)**: 외부 클리핑 자료, 사용자 개인 노트, 북마크 링크 등. 이 레이어는 절대 수정되지 않는 단일 진실 공급원(SSOT, Single Source of Truth)입니다.
2. **The Wiki (위키)**: LLM이 전적으로 소유하고 수정하는 마크다운 파일들의 디렉토리. 엔티티 페이지, 주제 요약, 지식 간의 연결 링크 구조가 이곳에 축적됩니다.
3. **The Schema (작업 규칙)**: `AGENTS.md`와 같이 LLM에게 위키의 레이아웃, 인덱스 생성 기준, 충돌 해결 및 작업 흐름을 통제하는 규칙서입니다. LLM이 단순 챗봇이 아닌 규율 있는 위키 관리자로 행동하게 만드는 중추 역할을 합니다.

---

## 3. 핵심 동작 (Operations)

### 3-1. Ingest (정보 섭취)
* **역할**: 사용자가 원본 폴더에 새 파일을 추가하고 Ingest 명령을 내리면, LLM은 이를 읽어 기존 위키와의 겹침 여부를 확인합니다.
* **처리 흐름**: 주요 키워드와 개념을 추출하고, 기존의 위키 개체/개념 페이지를 찾아 내용을 추가·수정하며, 새로운 링크(`[[링크]]`)를 걸어 연결 구조를 강화합니다. 작업 완료 후 `wiki/index.md`와 `wiki_Schema/Log/log.md`를 업데이트합니다.

### 3-2. Query (지식 탐색)
* **역할**: 누적된 위키 지식을 기반으로 자연어 질의에 답합니다.
* **복리 효과**: 질의를 통해 도출된 새로운 통찰, 비교 표, 상세 분석 등은 단순 채팅 내역으로 흘려보내지 않고 **위키에 새 페이지로 등록하여 다시 지식의 일부로 축적**시킵니다.

### 3-3. Lint (품질 검사 및 관리)
* **역할**: 정기적으로 위키의 건강 상태를 점검하고 가드닝(Gardening)을 진행합니다.
* **수행 작업**:
  * 서로 모순되는 주장이나 데이터가 존재할 경우 표시 및 대조
  * 아무 곳에서도 가리키지 않는 고립된 페이지(Orphan Page)에 대한 링크 연결
  * 중요 단어로 언급되었으나 개별 페이지가 부재한 개념 식별 및 신규 스캐폴딩 제안

## 4. 내비게이션 전용 파일

지식의 규모가 커짐에 따라 LLM과 사용자가 위키의 지도를 그릴 수 있게 돕는 특별한 파일들이 존재합니다.

* **`wiki/index.md` (콘텐츠 지향)**: 위키 내부의 모든 페이지 목록을 개념(Concepts), 도구(Tools), 메타(Meta), 원본 출처(Sources) 등으로 범주화하여 한 줄 요약 및 링크와 함께 제공합니다. LLM이 질문을 받았을 때 가장 먼저 읽고 상세 정보 탐색을 위한 진입점으로 삼는 탐색 지도입니다.
* **`wiki_Schema/Log/log.md` (시간순 이력 지향)**: 인제스트, 질의, 린트 등의 모든 활동을 날짜와 함께 기록하는 추가 전용(Append-only) 로그 파일입니다. `grep` 등의 Unix 명령어나 정규식을 통해 최근 작업 이력을 손쉽게 필터링할 수 있는 일정한 형식을 유지합니다.
  * 예시 형식: `## [2026-06-14] ingest | Clippings & raw_MyNote`

---

## 5. 도구 연동 및 최적화 팁

* **Obsidian (프론트엔드 IDE)**: 옵시디언은 인간이 지식을 입력하고 조회하기 위한 최고의 사용자 인터페이스(UI)입니다. 데이터의 주권이 로컬 컴퓨터의 파일 시스템에 보장되며, **그래프 뷰(Graph View)**를 통해 지식 간의 실제 연결망을 시각적으로 직관하기 좋습니다.
* **Obsidian Web Clipper**: 브라우저 확장 도구를 활용하여 인터넷 기사, 유튜브 영상 대본(Transcript) 등을 마크다운 형태로 원본 디렉토리에 손쉽게 수집할 수 있습니다.
* **지식 보관함(Vault) 분리**: 관심사(예: 자동차 정보와 웹 기술 정보)가 혼잡하게 섞이면 LLM의 쿼리 시 컨텍스트 혼선으로 신뢰도가 하락합니다. 관심사별로 옵시디언 볼트를 독립적으로 분리하여 관리하는 것이 성능 향상에 유리합니다.
